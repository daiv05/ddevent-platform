# Feature Documentation Index

This directory stores implementation notes for each feature worked in Devent.

## Rules

- Create one file per feature/work item.
- File naming format: `YYYY-MM-DD-feature-name.md`.
- Keep entries short but complete.
- Update the document in the same cycle as code changes.

## Required Sections

Every feature file should include:

1. Goal and Scope
2. Design Decisions
3. Impacted Areas (API/worker/dashboard/packages)
4. Contracts and Data Changes
5. Security and Multitenancy Notes
6. Observability Notes
7. Validation and Testing
8. Status and Next Steps

## Example

- `2026-03-16-event-ingestion-mvp.md`
